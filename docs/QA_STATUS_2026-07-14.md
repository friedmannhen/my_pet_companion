## Bugs found & fixed in this pass

1. **Ghost room presence (race)** — leaving the room while `join()` was
   still waiting to free a stale observer channel let the deferred
   subscribe run anyway, tracking presence in a room the UI had left.
   Fixed with a join-generation guard (`useRoom.ts`).
2. **Friends: declined request permanently blocked re-requesting** — the
   declined row shares the PK with any new insert, and RLS forbids the
   requester updating it. `request()` now deletes its own declined row
   first (`useFriends.ts`).
3. **History logged failures as successes** — "Created/Deleted/Left group"
   entries were written even when the RPC failed (offline, RLS).
   `leave`/`deleteGroup` now return success booleans and SideDock gates all
   group history logging on the actual result.
4. **RPS invite allowed during a Target Toss lobby/game** — now mutually
   exclusive in both directions.

## Known issues (accepted for now, ranked)

1. **Realtime channel auth unhardened** *(pre-existing, highest-value fix
   before going beyond friends-scale)* — channel topics embed raw group
   UUIDs; anyone told an id could subscribe and spoof presence/broadcasts.
   Also means minigame/battle events are trust-based between clients.
2. **Turn-skip race (low probability)** — a Target Toss throw sent right at
   the 15s AFK boundary can cross with the authority's skip; both carry the
   same `seq`, so clients keep whichever arrived first → possible desync.
   Needs presence flap + exact timing. Future fix: totally ordered event
   chain (hash/ack) or a lightweight resync broadcast.
3. **Deleting a group doesn't kick live members** — their channel survives;
   they idle in a ghost room until they leave. Needs a room-closed
   broadcast or #1.
4. **Ties in `minigame_scores` are indistinguishable from losses**
   (games_played+1, games_won+0; no ties column). By-schema; fine until a
   win-rate UI exists.
5. **Egg cosmetics are placeholders** — the picker's 3 eggs are tints; the
   chosen tint isn't persisted and gameplay is always `cat`. Blocked on art
   for more `PetType`s.
6. **Marker-before-landing** — a Target Toss landing marker appears the
   instant the event applies, ~1s before the replayed ball visually lands.
7. **localStorage save write is undebounced** — every action serializes the
   full save (incl. ≤200 history entries). No observed jank; if it appears,
   debounce the write like the cloud push (2s).
8. **Quest sync is partial cross-device** (`rewardHistory`, `cap`,
   `lastQuestMenuOpenedAt` reset on device switch) and the
   `quest_reward_history` DB table has no writers — decide to wire or drop.
9. **Duplicate-direction friends rows possible** (A→B declined + B→A
   pending shows twice) — cosmetic at dev scale.

## Multi-user test checklist (what to actually exercise)

Setup: 2–3 test accounts, two browser-mock tabs (pet-game-quests-testing
skill technique) or two machines. **Run `npx supabase db push` first.**

- [ ] Fresh signup → egg picker appears once → warm mode (lamp → glow
      cursor → hold on egg) → hatch → scale springs to 0.7 after jump-out.
- [ ] Room: presence count in groups menu (expanded row) matches reality
      while a second account joins/leaves; positions/chat/emotes flow.
- [ ] Battle: challenge → accept → identical result both sides; history
      entries show clamp-aware deltas.
- [ ] RPS: 🎮 invite → both pick → identical outcome; tie path; abandon ✕;
      `minigame_scores` rows update (needs db push).
- [ ] Target Toss: lobby cap turn-away (3rd account, cap 2) → ready → start
      → 3 rounds round-robin → one deliberate 15s AFK skip → one client
      closes mid-game (auto-skips continue) → forced tie → sudden death →
      winner banner → scores recorded once per participant.
- [ ] Host leaves lobby pre-start → lobby auto-closes on other clients.
- [ ] Groups: owner delete (confirm inline) → disappears everywhere after
      refresh; non-owner cannot see delete; Global undeletable.
- [ ] Friends: search → request → accept on 2nd account (badge shows) →
      remove → re-request after a decline (regression for fix #2).
- [ ] Same machine, account switch: no save/history bleed (SAVE_OWNER_KEY),
      no 460-carePoints-style seeding.
- [ ] Windows taskbar icon stays hidden when settings/groups/friends focus
      a text input (real Electron run, not browser mock).
